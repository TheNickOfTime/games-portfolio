Pac Man Level Editor: Part 2

NickOctober 5, 2017October 5, 2017

I thought I was done with this project, but I was given another week to work on it. I used this time to move the editor from being attached to a script to having its own editor window, as well as improve some functionality. Some of these improvements include the ability to resize the grid, place teleporters, and buttons now use colored icons rather than characters (this helps while building large levels tremendously. There are plenty of other insignificant changes, but overall I am very happy with how this ended up turning out. The script that comprises the level editor is one of the more complicated things I have written to date, and I have learned an invaluable amount of information in creating this.

Finished up the final prototype for my one button game. I am really happy with this prototype, given the short amount of time I had to work on it. Pac Man – Game & Level Editor: Part 1

NickSeptember 24, 2017October 5, 2017

As a warm-up assignment for one of my classes, we were tasked with recreating Pac Man from scratch in Unity. It was a fun task, there are lots of nuances to the mechanics of Pac Man, so there were lots of things to do. My Enemy AI is definitely nothing fancy, the ghosts just choose a random valid direction to go in when they near a wall. I have little experience writing AI so I am still quite happy with how this aspect turned out.

More interestingly, the next stage of the assignment was to create a level editor. We were told to create something that can read a text file that represents the map, but I wanted to take things to the next level. I had never dabbled in custom editor scripting, so I dove in head first to create a level editor out of editor GUI buttons. Clicking a button will cycle through available level elements, such as Pac Man, walls, Pac Dots, etc. I also included a level reset button for when you realize what you have made is no good. I learned a lot of valuable information about editor scripting, and I’m glad I gave myself this challenge!

Summer Laziness

NickJuly 31, 2017September 24, 2017

It has been a lazy/busy summer. In between classes I have been working on a master time system. To this, I have tied a day/night cycle and a variable temperature system. All of this works with the master time system and has the ability to go forward in time as well as backward. My initial purpose with this little project was to learn more about how the real-time lighting system works in Unity. Turns out it is pretty hard to make a custom time system that is a bit more simplistic than the one unity provides.

Some notes on the video:

  • Time moves forward when the character does
  • It is cooler in the shadows than in direct sunlight
